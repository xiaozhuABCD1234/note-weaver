# Note Weaver

AI 驱动的 Obsidian 笔记助手，支持智能问答、笔记修改和知识检索。

## 功能

- **AI 聊天助手** — 在 Obsidian 内直接与 AI 对话，支持流式输出
- **工具调用（Tool Calling）** — AI 可直接读取、创建、修改、删除、重命名库内笔记，以及搜索文件内容
- **RAG 知识检索** — 自动从 Vault 中检索与问题相关的笔记内容作为上下文
- **思考模式** — 支持 DeepSeek 等模型的内部推理（reasoning），提升复杂任务处理能力
- **选中文本修改** — 选中笔记中的文本，让 AI 直接修改
- **读取库内文件** — 指定文件路径，将内容发送给 AI 进行处理
- **操作日志** — 自动记录 AI 交互、工具调用等日志，便于调试和审计

## 安装

1. 在 Obsidian 中打开 **设置 → 社区插件**，关闭安全模式
2. 点击 **浏览**，搜索 "Note Weaver"
3. 点击 **安装**，然后 **启用**
4. 或者手动将 `main.js`、`manifest.json`、`styles.css` 复制到 `<Vault>/.obsidian/plugins/note-weaver/`

## 配置

在 **设置 → Note Weaver** 中配置以下参数：

| 参数       | 说明                                 | 默认值                     |
| ---------- | ------------------------------------ | -------------------------- |
| API Key    | LLM API 密钥（支持 OpenAI 兼容接口） | —                          |
| Base URL   | API 服务器地址                       | `https://api.deepseek.com` |
| Model      | 模型名称                             | `deepseek-v4-flash`        |
| Max Tokens | 每次请求的最大 Token 数              | `16384`                    |
| 思考模式   | 启用后模型先进行内部推理再输出       | 开启                       |
| 思考强度   | 推理深度（high / max）               | `high`                     |

### RAG 设置

| 参数         | 说明                       | 默认值     |
| ------------ | -------------------------- | ---------- |
| 启用 RAG     | 开启后 AI 自动检索相关笔记 | 关闭       |
| 最大检索块数 | 每次查询最多检索的片段数   | `5`        |
| 分块大小     | 每个笔记片段的最大字符数   | `1000`     |
| 分块重叠     | 相邻片段之间的重叠字符数   | `200`      |
| 检索范围     | 当前文件夹 / 整个 Vault    | 整个 Vault |

## 使用

### 命令

| 命令                    | 说明                                   |
| ----------------------- | -------------------------------------- |
| AI: 修改选中文本        | 将编辑器中选中的文本发送给 AI 进行处理 |
| 读取库内文件并发送给 AI | 输入文件路径，将文件内容发送给 AI      |
| 重建知识索引            | 重新扫描 Vault 构建 RAG 索引           |

### 聊天界面

点击左侧 Ribbon 栏的机器人图标或执行任意 AI 命令即可打开聊天面板。

- 输入消息回车发送
- Esc 取消正在进行的 AI 回复
- AI 会自动调用工具读写笔记，并在操作后显示预览

## 开发

```bash
# 安装依赖
npm install

# 开发模式（监听文件变化）
npm run dev

# 生产构建
npm run build
```

构建产物为 `main.js`，复制到插件目录即可。
